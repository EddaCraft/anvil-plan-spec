# APS v0.3 Implementation Plan - Summary

**Status:** Draft - Ready for Review
**Created:** 2026-01-15
**Scope:** Complete implementation of usability improvements (everything except IDE integration)

---

## What's in This Plan

This is a comprehensive plan to make APS production-ready by addressing:

1. **Machine discoverability** for AI agents
2. **Complete validation tooling**
3. **Zero-friction npm onboarding**
4. **Terminal dashboard** for progress visibility
5. **AI agent reliability testing**
6. **Enhanced templates and documentation**

## Modules Overview

### 01. Validation (VAL) - Already Planned ✓
**Status:** Ready (from v0.2)
**Work Items:** 6 (VAL-001 through VAL-006)
**Deliverable:** `aps lint` CLI command for validating APS documents

Key features:
- Check required fields present
- Validate ID formats
- Check dependency references
- Warn on common issues
- CI integration example

---

### 02. npm Init (INIT) - Already Planned ✓
**Status:** Ready (from v0.2)
**Work Items:** 10 (INIT-001 through INIT-010)
**Deliverable:** `npm init aps` interactive setup

Key features:
- Interactive prompts for project setup
- Scaffold generation with templates
- `--update` flag for refreshing templates
- Non-interactive mode for CI/CD
- Published to npm as primary installation method

---

### 03. Core Infrastructure (CORE) - New 🆕
**Status:** Draft
**Work Items:** 9 (CORE-001 through CORE-009)
**Deliverable:** `.aps/config.json` and frontmatter support

Key features:
- Machine-readable config at `.aps/config.json`
- YAML frontmatter in templates
- JSON schemas for validation
- Config discovery and parsing utilities
- Version tracking for template upgrades

**Why this matters:**
- AI agents can programmatically discover APS structure
- Tooling can parse APS documents reliably
- Future features enabled (custom validators, alternative layouts)

---

### 04. Enhanced Templates (ETPL) - New 🆕
**Status:** Draft
**Work Items:** 10 (ETPL-001 through ETPL-010)
**Deliverable:** Templates with inline examples and visual markers

Key features:
- Inline mini-examples in all templates
- Visual markers (✏️ REQUIRED, 🔌 OPTIONAL)
- Example-driven guidance
- Before/after template pairs
- Field-by-field tooltips

**Why this matters:**
- Users see what a filled template looks like
- Reduced cognitive load (clear required vs optional)
- Faster onboarding with realistic examples

---

### 05. Enhanced Documentation (EDOC) - New 🆕
**Status:** Draft
**Work Items:** 10 (EDOC-001 through EDOC-010)
**Deliverable:** Complete docs with AI decision trees and glossary

Key features:
- Complete terminology migration audit
- AI agent decision tree in aps-rules.md
- Before/after examples in prompting docs
- Comprehensive glossary
- Troubleshooting guide
- Solo developer guidance
- npm init as hero of README

**Why this matters:**
- AI agents know what to do without asking
- Users find answers quickly
- Consistent terminology prevents confusion

---

### 06. TUI Dashboard (TUI) - New 🆕
**Status:** Draft
**Work Items:** 12 (TUI-001 through TUI-012)
**Deliverable:** `aps status` terminal dashboard

Key features:
- Module progress visualization
- Work item status breakdown
- Dependency graph (text-based)
- Interactive navigation
- Watch mode (auto-refresh)
- Summary mode for CI/CD

**Why this matters:**
- Quick status check without leaving terminal
- Visual progress tracking
- Identify blockers immediately
- Web dashboard deferred to long-term roadmap

---

### 07. AI Testing (AI) - New 🆕
**Status:** Draft
**Work Items:** 11 (AI-001 through AI-011)
**Deliverable:** Test framework for verifying agent compliance

Key features:
- Test scenario library (markdown)
- Evaluation scorer (deterministic)
- Fixture-based tests for CI
- Agent compliance matrix
- Anti-pattern detection
- Live API testing (optional)

**Why this matters:**
- Verify aps-rules.md actually works
- Regression testing when guidance changes
- Track compliance across agents (Claude, GPT-4, Cursor)
- Identify where agents struggle

## Execution Strategy

### Phase 1: Foundation (Week 1)
**Goal:** Unblock everything else

- Complete VAL module (validation CLI)
- Complete INIT module (npm package)
- Publish `create-aps` to npm

**Deliverables:**
- `aps lint` command works
- `npm init aps` works
- Package published to npm

---

### Phase 2: Core Infrastructure (Week 1-2)
**Goal:** Enable machine discoverability

- Complete CORE module
- Create `.aps/config.json` spec
- Add frontmatter to templates
- Create JSON schemas

**Deliverables:**
- Config spec documented
- Frontmatter examples in templates
- Schemas validate correctly

---

### Phase 3: Enhanced Content (Week 2)
**Goal:** Improve usability

- Complete ETPL module (template improvements)
- Complete EDOC module (documentation)
- Update README with npm init as hero

**Deliverables:**
- All templates have inline examples
- Terminology migration complete
- AI decision tree in aps-rules.md
- Glossary comprehensive

---

### Phase 4: Tooling (Week 3)
**Goal:** Build visibility and reliability tools

- Complete TUI module (dashboard)
- Complete AI module (testing)

**Deliverables:**
- `aps status` command works
- Agent test suite passes
- Compliance matrix shows >80% average

---

### Phase 5: Polish (Week 4)
**Goal:** Release-ready state

- Integration testing
- Documentation review
- Blog post / announcement
- GitHub polish (badges, screenshots)

**Deliverables:**
- All modules Complete
- README polished
- Release notes written
